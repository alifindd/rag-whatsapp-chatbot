# RAG Customer Service Chatbot — WhatsApp + FastAPI + LLM

A production-ready customer service chatbot that answers questions from your own data using a custom RAG (Retrieval-Augmented Generation) pipeline. Integrated with WhatsApp via Twilio.

---

## Demo

> Customer sends a WhatsApp message → Bot retrieves relevant FAQ context → LLM generates a friendly, natural response → Reply sent back to WhatsApp

![Demo](images/demo_chat.png)

---

## Features

- **Custom RAG Pipeline** — semantic search over your FAQ data using sentence embeddings + cosine similarity
- **WhatsApp Integration** — real-time responses via Twilio WhatsApp API
- **LLM-Powered Responses** — natural, friendly replies generated by Groq LLM
- **REST API** — clean FastAPI backend with Pydantic validation
- **Modular Architecture** — routers, schemas, and services fully separated

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| API Framework | FastAPI + Pydantic |
| Embedding Model | Sentence Transformers (`all-MiniLM-L6-v2`) |
| Similarity Search | Scikit-learn Cosine Similarity |
| LLM | Groq API (OpenAI-compatible) |
| WhatsApp | Twilio WhatsApp API |
| Tunneling (dev) | ngrok |

## How It Works

```
User Message (WhatsApp)
        ↓
Twilio Webhook → POST /chat/whatsapp
        ↓
Embed user question (Sentence Transformers)
        ↓
Cosine similarity search over FAQ embeddings
        ↓
Retrieve top-3 most relevant answers
        ↓
Pass context + question to Groq LLM
        ↓
LLM generates natural, friendly response
        ↓
Return TwiML response → WhatsApp reply
```

## Getting Started

### Prerequisites
- Python 3.9+
- Twilio account (free sandbox available)
- Groq API key (free tier available)
- ngrok (for local development)

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/alifindd/rag-whatsapp-chatbot.git
cd rag-whatsapp-chatbot
```

2. **Create virtual environment**
```bash
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Mac/Linux
```

3. **Install dependencies**
```bash
pip install -r requirements.txt
```

4. **Setup environment variables**

Create a `.env` file in the root directory:
```env
GROQ_API_KEY=your_groq_api_key_here
```

5. **Run the server**
```bash
uvicorn app.main:app --reload
```

6. **Expose local server with ngrok**
```bash
ngrok http 8000
```

7. **Configure Twilio Sandbox**
   - Go to [Twilio Console](https://console.twilio.com) → Messaging → Try it out → Send a WhatsApp message → Sandbox Settings
   - Set **"When a message comes in"** to:
   ```
   https://YOUR-NGROK-URL/chat/whatsapp
   ```
   - Method: `HTTP POST`

8. **Test it!**

Send a message to your Twilio Sandbox WhatsApp number and the bot will reply. 🎉

## 📊 RAG Pipeline Details

- **Embedding Model**: `all-MiniLM-L6-v2` — lightweight, fast, and accurate for semantic search
- **Similarity Metric**: Cosine similarity
- **Top-K Retrieval**: Returns top 3 most relevant FAQ answers as context
- **LLM Temperature**: 0.5 — balanced between creativity and consistency

---

## 🔧 Customization

### Using your own FAQ data
Replace `app/data/apparel_store_faq.csv` with your own CSV file. Required columns:
```
question, answer
```

### Changing the LLM behavior
Edit the system prompt in `app/services/rag_service.py` → `generate_answer()` to match your brand's tone and rules.
